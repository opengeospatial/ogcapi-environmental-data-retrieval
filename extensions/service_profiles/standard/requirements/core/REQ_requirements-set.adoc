[[req_core_requirements-set]]

NOTE: A Service profile can apply to multiple `Collections` and an OGC API EDR implementation can support multiple `Service Profiles`.


[requirement]
====
[%metadata]
identifier:: /req/core/requirements-set
statement:: 

part:: The collection SHALL clearly identify the service profile that applies.

part:: The collection SHOULD provide a link to the service profile definition with a link relation type of `profile`.

part:: The service profile SHALL consist of a set of requirements for a `Collection` and (if the `Collection` supports `Instances`) the `Instances` of the `Collection`. For each of the attributes listed, if it is in the `Collection` (or instance), there SHALL be a requirement to define it.

====



[permission]
====
[%metadata]
identifier:: /per/core/multiple-service-profile-definition
statement:: An OGC API EDR implementation MAY include multiple service profiles.

====
